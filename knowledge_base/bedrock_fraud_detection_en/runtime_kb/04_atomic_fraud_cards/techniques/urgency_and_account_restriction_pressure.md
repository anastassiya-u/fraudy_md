# Urgency and account restriction pressure

## Pattern summary
Attackers use deadlines, account blocks, fines, or loss threats to prevent verification.

## Core deception
The user is led to believe immediate action is safer than pausing to verify.

## Common channels
- sms
- email
- phone_call
- messenger
- web_page

## Red flags
- Act now/final notice language
- Threat of account suspension, fine, arrest, or package return
- Caller discourages hanging up or talking to others
- Urgency paired with payment, login, OTP, or install request

## Scam examples
- Final notice: verify now or account will be blocked.
- Do not hang up; your money is at risk.

## Legitimate contrast
Legitimate alerts can be urgent, but they allow verification through official channels and do not require secrets by phone/chat.

## What NOT to classify as fraud automatically
- User can verify by opening official app manually
- Message contains no request for secrets, payment, or installation

## Safe user actions
- Pause before acting
- Use official app/site/phone number
- Do not share secrets under time pressure
- Ask a trusted person if unsure

## Runtime model instruction
Use this card as a pressure-tactic signal. Urgency alone is medium risk; urgency plus sensitive requested action is high/critical.
