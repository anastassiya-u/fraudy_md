# OTP code theft scam

## Pattern summary
An attacker asks the user to read, forward, screenshot, or enter a one-time code outside the official authentication flow.

## Core deception
The user is led to believe sharing the code is necessary for verification, cancellation, delivery, refund, or security.

## Common channels
- phone_call
- sms
- messenger
- email
- web_page

## Red flags
- Any person asks for OTP, MFA code, or push approval
- Caller keeps user on line during login
- Code is requested to cancel a transaction or protect account
- Code arrives unexpectedly

## Scam examples
- Read me the code to block the transaction.
- Send the verification code so we can confirm delivery.

## Legitimate contrast
Legitimate services may send OTPs, but the code is for the user to enter only in the official app/site they initiated.

## What NOT to classify as fraud automatically
- User requested login/reset and enters code only in official app
- No human asks to hear or receive the code

## Safe user actions
- Never share OTP/MFA codes
- Deny unexpected push approvals
- Stop the call/chat
- Change password and contact service if code was shared

## Runtime model instruction
Use this card as a strong escalation signal. OTP sharing requests are at least high risk and often critical when bank/payment accounts are involved.
