# Fake support remote access scam

## Pattern summary
A fake support alert, search result, or caller claims the device/account is infected and requests remote access or payment.

## Core deception
The user is led to believe urgent technical help is needed and remote control is safe.

## Common channels
- phone_call
- web_page
- email
- messenger

## Red flags
- Pop-up says device is infected
- Support number appears in browser/ad
- Remote access or screen sharing requested
- Payment requested to fix issue
- User guided into banking or password screens

## Scam examples
- Your device is infected; call support immediately.
- Install remote support app so we can secure your account.

## Legitimate contrast
Legitimate support does not appear as an unsolicited browser panic message and should not require banking access during screen sharing.

## What NOT to classify as fraud automatically
- User contacted official support through known account portal
- No remote access, payment, or password screen sharing requested

## Safe user actions
- Close the page or disconnect internet if needed
- Do not grant remote access
- Contact official support independently
- Change passwords if access was granted

## Runtime model instruction
Use this card when support framing combines infection/account danger with remote access, payment, or banking guidance. Treat as high/critical.
